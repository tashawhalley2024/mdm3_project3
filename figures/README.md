The focal predictor `FOCAL_PRED = composite_secularism_norm` (Item 2 redux, rebuilt 2026-04-16) is an equal-weight z-score composite over three *structural* Pew GRI sub-items (state religion, religious law, religious courts) and four WVS religious-intensity items — seven inputs across two dimensions. Built in-memory at load time by `analysis/utils.py:build_secularism_composite`; not stored on disk. An earlier version had 11 inputs across 3 dimensions; four treatment items (`gri_apostasy_norm`, `gri_blasphemy_norm`, `gri_gov_favour_norm`, `v2clrelig_norm`) were removed from the composite on circularity grounds and now appear as standalone sub-item focals in the decomposition. `FOCAL_PRED_2 = gri_apostasy_norm` is the secondary sub-item focal (strongest individual GRI component). `FOCAL_PRED_LEGACY = gri_religious_courts_norm` is preserved as a robustness focal with outputs suffixed `_religious_courts`; it does not drive the figure set.

**Removed from the presentation set:**
- `03_coefplot_suboutcomes.png` — underlying Phase 3 sub-outcome tiers are no longer in `results.csv`.
- `04_trend.png` — redundant with `12_mundlak_decomposition.png`, which makes the structural vs temporal argument more rigorously.

- Oster sensitivity for the composite and legacy courts focals — applying Oster to a broad composite with multiple dimensions or to a known null sub-item is not informative; only the apostasy version is retained as the Oster headline.

## Regenerating

```bash
python analysis/run_plots.py
```

### Palette mode

All figures use a two-tier significance palette by default (red for p<0.05, grey
otherwise — matches the slide deck). Set `PLOTS_PRES_MODE=0` to use the
four-tier writeup palette (p<0.01 / p<0.05 / p<0.10 / n.s.):

```bash
PLOTS_PRES_MODE=0 python analysis/run_plots.py   # writeup build
PLOTS_PRES_MODE=1 python analysis/run_plots.py   # slide build (default)
```

The toggle is read once at module load via
`PRES_MODE = os.environ.get("PLOTS_PRES_MODE", "1") == "1"` and cascades to
every figure through `_sig_colour` and `_sig_legend_handles`.
